## 2. Domain Feature Extraction — `src/features/domain_features.py`

### Shannon Entropy
```python
# src/features/domain_features.py  (line 139-145)
def _calculate_entropy(self, text: str) -> float:
    prob = [float(text.count(c)) / len(text) for c in set(text)]
    entropy = -sum([p * math.log2(p) for p in prob])
    return round(entropy, 4)
```

Entropy measures **randomness**. The domain `xk9f2mz.tk` has very high entropy
(random-looking characters) while `google.com` has low entropy (common letters).
Phishing domains generated by algorithms tend to have high entropy.

- `set(text)` gives unique characters.
- The formula `-sum(p * log2(p))` is Claude Shannon's information entropy formula.
- A higher result = more random-looking domain name.

### WHOIS Date Parsing
```python
# src/features/domain_features.py  (line 97-128)
def _calc_days_diff(self, date1, date2, reverse=False):
    if isinstance(date1, list):
        date1 = date1[0]      # WHOIS sometimes returns a list; take the first
    if isinstance(date1, str):
        date1 = datetime.fromisoformat(date1)  # Parse "2024-01-15" → datetime object
    # Strip timezone info to avoid "offset-aware vs naive" Python errors
    if hasattr(date1, 'tzinfo') and date1.tzinfo is not None:
        date1 = date1.replace(tzinfo=None)
    diff = (date2 - date1).days if not reverse else (date1 - date2).days
    return float(max(0, diff))
```

This is defensive programming. WHOIS data from different registrars can come back as
a `str`, `datetime`, or even a `list`. Each case is handled so no crash occurs.
`reverse=True` is used when calculating how old a domain is (today minus creation date).

## 3. Graph Feature Extraction — `src/features/graph_features.py`

```python
# src/features/graph_features.py  (line 57-59)
neighbors = list(graph.neighbors(node_id)) + list(graph.predecessors(node_id))
phishing_neighbors = [n for n in set(neighbors) if graph.nodes[n].get('label') == 'phishing']

```

- `graph.neighbors(node_id)` gives all nodes this domain **points to** (outgoing edges).
- `graph.predecessors(node_id)` gives all nodes **pointing at** this domain (incoming edges).
- The list comprehension `[n for n in ... if ...]` is Python's compact way of filtering a list.
- `.get('label') == 'phishing'` safely reads the node's label attribute (returns `None` instead
  of crashing if the attribute doesn't exist).

```python
# src/features/graph_features.py  (line 100-104)
my_trust = node_data.get('trust_score', 0.5)
cluster_trusts = [graph.nodes[n].get('trust_score', 0.5) for n in cluster_nodes]
```

The anomaly score measures how **different** a domain is from its cluster average.
A domain with trust 0.9 sitting in a cluster with average trust 0.1 is anomalous — it
may be a compromised legitimate site.

## 4. Trust Propagation — `src/graph/trust_propagation.py`

```python
# src/graph/trust_propagation.py  (line 26-33)
for node, data in graph.nodes(data=True):
    label = data.get('label', 'unknown')
    if label == 'safe':
        trust_scores[node] = 1.0
    elif label == 'phishing':
        trust_scores[node] = 0.0
    else:
        trust_scores[node] = 0.5  # Unknown nodes start neutral
```

```python
# src/graph/trust_propagation.py  (line 36-66)
for i in range(iterations):           # Repeat 10 times
    for node, data in graph.nodes(data=True):
        if label in ['safe', 'phishing']:
            continue                   # Anchors never change
        # Weighted average of all neighbors' trust scores
        weighted_avg = neighbor_sum / total_weight
        # Blend old score with neighbor influence (damping_factor = 0.3)
        new_scores[node] = (1 - 0.3) * trust_scores[node] + 0.3 * weighted_avg
```

The `damping_factor` (0.3) controls how much neighbors influence you each iteration.
At 0.3, you keep 70% of your own score and absorb 30% from neighbors.
After 10 iterations, trust scores converge to stable values — an established domain
linked to many safe sites will score close to 1.0.

## 5. Data Preprocessing — `src/data/preprocessor.py`

```python
# src/data/preprocessor.py  (line 16)
self.imputer = SimpleImputer(strategy='constant', fill_value=-1)
self.scaler = StandardScaler()
```

**Imputation** replaces missing values (NaN) with a constant (-1) so the ML model
isn't confused by gaps. Strategy `'constant'` means all NaN values become the same number.

**StandardScaler** transforms every feature so it has mean=0 and standard deviation=1.
Without this, a feature like `url_length` (range: 10–500) would dominate features like
`has_ip_address` (range: 0–1). Scaling puts them on equal footing.

```python
# src/data/preprocessor.py  (line 42-50)
def transform(self, X):
    X_imputed = self.imputer.transform(X)   # Fill NaNs with -1
    X_scaled  = self.scaler.transform(X_imputed)  # Normalize
    return X_scaled
```

Note: in production (inference), we call `.transform()` not `.fit_transform()`.
`fit_transform` learns the statistics from data (used during training).
`transform` applies those already-learned statistics (used at prediction time).


Note that TGIS is inverted: `(1 - tgis_score)`. A tgis_score of 0.9 means "very trusted",
so its phishing contribution is `1 - 0.9 = 0.1` (low phishing risk).

## 7. Database Models — `api/models.py`

```python
# api/models.py  (line 8-25)
class Prediction(Base):
    __tablename__ = "predictions"
    id         = Column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    url        = Column(Text, nullable=False, index=True)
    created_at = Column(DateTime(timezone=True), server_default=func.now())
    feature_vector = relationship("FeatureVector", back_populates="prediction", uselist=False)
```

- `UUID` is a universally unique identifier (e.g., `"a7b3c2d1-..."`). Safer than auto-increment integers.
- `default=uuid.uuid4` means Python generates the ID automatically before inserting.
- `index=True` on `url` creates a database index, making `WHERE url = '...'` queries fast.
- `server_default=func.now()` tells the database to set the timestamp (not Python — more accurate).
- `relationship(uselist=False)` means each Prediction has **one** FeatureVector (not a list).

```python
# api/models.py  (line 30-46)
class FeatureVector(Base):
    features = Column(JSONB, nullable=False)
```

`JSONB` is PostgreSQL's binary JSON type. Unlike plain `JSON`, JSONB is indexed and can be
queried efficiently: `WHERE features->>'domain_age_days' < '90'`.

## 8. Database Session — `api/database.py`

```python
# api/database.py  (line 26-35)
def get_db():
    db = SessionLocal()
    try:
        yield db        # "yield" means: give the session to whoever asked for it
    finally:
        db.close()      # This runs even if an exception occurred (guaranteed cleanup)
```

`yield` makes this a Python **generator function**. FastAPI treats it as a context manager —
it gets the session before handling the request, and closes it after (no matter what).
This prevents database connection leaks.
